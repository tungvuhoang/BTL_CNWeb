1. Tổng quan dự án
1.1 Mục tiêu
Xây dựng web quiz realtime kiểu Kahoot, gồm 2 vai trò:
Host
• đăng ký / đăng nhập
• tạo quiz
• thêm / sửa / xóa câu hỏi
• tạo phòng chơi
• xem player join
• start game
• chuyển câu tiếp theo
• kết thúc game
• xem leaderboard
Player
• không cần đăng nhập
• nhập PIN + tên để vào phòng
• chờ host start
• nhận câu hỏi realtime
• chọn đáp án
• nhận điểm
• xem leaderboard
• reconnect được khi refresh

4. Database schema final
4.1 Bảng users
Column Type Constraint
id BIGINT PK, auto increment
username VARCHAR(100) unique, not null
password_hash VARCHAR(255) not null
created_at DATETIME not null
Ý nghĩa
Lưu tài khoản host.
4.2 Bảng quizzes
Column Type Constraint
id BIGINT PK, auto increment
host_id BIGINT FK -> users.id, not null
title VARCHAR(255) not null
created_at DATETIME not null
Quan hệ
• 1 user có nhiều quiz
• 1 quiz thuộc 1 host
4.3 Bảng questions
Column Type Constraint
id BIGINT PK, auto increment
quiz_id BIGINT FK -> quizzes.id, not null
content TEXT not null
answer_a TEXT not null
answer_b TEXT not null
answer_c TEXT not null
answer_d TEXT not null
correct_answer VARCHAR(1) not null
time_limit INT not null
created_at DATETIME not null
Rule
• correct_answer chỉ nhận: A, B, C, D
• time_limit > 0
4.4 Bảng game_rooms
Column Type Constraint
id BIGINT PK, auto increment
quiz_id BIGINT FK -> quizzes.id, not null
pin VARCHAR(20) unique, not null
Column Type Constraint
status VARCHAR(20) not null
current_question_index INT not null, default -1
started_at DATETIME null
ended_at DATETIME null
question_started_at DATETIME null
created_at DATETIME not null
Rule
• status chỉ nhận:
o waiting
o playing
o finished
• current_question_index = -1 nghĩa là chưa start
4.5 Bảng players
Column Type Constraint
id BIGINT PK, auto increment
room_id BIGINT FK -> game_rooms.id, not null
name VARCHAR(100) not null
score INT not null, default 0
session_id VARCHAR(255) null
is_connected BOOLEAN not null, default true
last_seen_at DATETIME null
Column Type Constraint
joined_at DATETIME not null
Unique
• (room_id, name)
Ý nghĩa
Trong cùng 1 room không cho 2 người trùng tên.
4.6 Bảng player_answers
Column Type Constraint
id BIGINT PK, auto increment
room_id BIGINT FK -> game_rooms.id, not null
player_id BIGINT FK -> players.id, not null
question_id BIGINT FK -> questions.id, not null
selected_answer VARCHAR(1) not null
is_correct BOOLEAN not null
points_earned INT not null, default 0
answered_at DATETIME not null
Unique
• (room_id, player_id, question_id)
Rule
• 1 player chỉ được submit 1 lần cho 1 question trong 1 room

6. Business rules final
6.1 Auth
Host
• phải login
• dùng JWT
• được phép:
o quản lý quiz
o quản lý question
o tạo room
o start / next / end game
Player
• không cần login
• chỉ join bằng:
o pin
o name
6.2 Quiz ownership
Mọi API host thao tác với quiz hoặc room phải check:
• quiz đó thuộc host đang login
• room đó được tạo từ quiz của host đang login
Nếu không đúng:
• trả lỗi QUIZ_FORBIDDEN hoặc AUTH_FORBIDDEN
6.3 Start game
Điều kiện
• room đang waiting
• quiz có ít nhất 1 question
Khi start
• status = playing
• currentQuestionIndex = 0
• startedAt = now
• questionStartedAt = now
Event broadcast
• GAME_STARTED
• QUESTION_CHANGED
6.4 Current question
Question hiện tại được xác định bởi:
• currentQuestionIndex
Player được thấy
• questionId
• content
• answerA
• answerB
• answerC
• answerD
• timeLimit
• timeRemaining
• questionNumber
• totalQuestions
Player không được thấy
• correctAnswer
6.5 Timer
Server lưu
• questionStartedAt
• timeLimit
Client tự tính
timeRemaining = timeLimit - (currentTime - questionStartedAt)
Rule
Nếu timeRemaining <= 0
• coi như hết giờ
• không cho submit nữa
Ghi chú
Server không cần push countdown mỗi giây.
6.6 Submit answer
Hợp lệ khi
• room đang playing
• player thuộc room đó
• questionId gửi lên đúng là current question
• player chưa submit câu đó
• chưa hết giờ
Khi submit
• nếu đúng:
o isCorrect = true
o pointsEarned = 100
• nếu sai:
o isCorrect = false
o pointsEarned = 0
Sau đó:
• lưu vào player_answers
• cộng điểm vào players.score
Event broadcast
• LEADERBOARD_UPDATED
6.7 Next question
Chỉ host được gọi
Logic
• nếu room không ở playing → reject
• nếu còn câu tiếp theo:
o currentQuestionIndex += 1
o questionStartedAt = now
o broadcast QUESTION_CHANGED
• nếu hết câu:
o status = finished
o endedAt = now
o broadcast GAME_ENDED
Không cần chờ tất cả player trả lời
Host có thể bấm next thủ công.
6.8 End game
Chỉ host được gọi
Khi end
• status = finished
• endedAt = now
Event broadcast
• GAME_ENDED
6.9 Reconnect
Player refresh
Frontend phải:
1. đọc localStorage:
o playerRoomId
o playerId
o playerName
2. gọi REST snapshot:
o room state
o current question
o leaderboard
3. reconnect websocket
Backend luôn phải check
• playerId có thuộc roomId không

14. Realtime flow final
14.1 Player join
1. Player gọi REST POST /join
2. Server tạo player record
3. Server broadcast PLAYER_JOINED
14.2 Host start
1. Host gọi REST POST /start
2. Server update DB
3. Server broadcast:
o GAME_STARTED
o QUESTION_CHANGED
14.3 Player submit
1. Player gọi REST POST /submit-answer
2. Server validate + lưu DB + cộng điểm
3. Server broadcast LEADERBOARD_UPDATED
14.4 Host next
1. Host gọi REST POST /next
2. Server update current question
3. Server broadcast QUESTION_CHANGED
4. Nếu hết câu thì broadcast GAME_ENDED
14.5 Host end
1. Host gọi REST POST /end
2. Server update status
3. Server broadcast GAME_ENDED
